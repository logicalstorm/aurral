# Aurral v2

v2 is a major rewrite. Hundreds of improvements, but here is what you will actually notice.

## What changed for v1 users

| v1                                                | v2                                                              |
| ------------------------------------------------- | --------------------------------------------------------------- |
| `/app/backend/data` mount                         | `/config` mount (old path auto-detected)                        |
| Alpine Docker image                               | Debian Bookworm (sharp and Honker need it)                      |
| "Weekly Flow" everywhere                          | "Playlists" - flows still exist as the dynamic kind             |
| Requests + Downloads + History pages              | One Activity page with Queue, Review, and History tabs          |
| Settings as one long page                         | Tabbed settings with test-before-save on every integration      |
| Custom Soulseek client built into Aurral          | You run slskd alongside Aurral, Aurral orchestrates it          |
| Spotify import = export CSV, convert, import JSON | In-app OAuth on the Playlists page, connect and pick a playlist |
| Blocklist page                                    | Discovery feedback (thumbs-up/thumbs-down)                      |

## New in v2

### Playlists

- **Spotify import** - connect in-app, pick a playlist, optional auto-sync
- **Lidarr import list feeds** - each flow generates a URL Lidarr can poll as a custom import list
- **Plex sync** - Aurral creates an Aurral music library in Plex and builds native Plexamp playlists via API
- **File reuse** - link existing completed tracks instead of re-downloading
- **Deep Dive** - reaches into less popular catalog when you want more adventurous flows

### Downloads

- **Usenet support** - Prowlarr + SABnzbd/NZBGet as an alternate or fallback download source
- **Source priority** - Aurral tries sources in order (slskd first by default, or Usenet first if you swap)
- **Track review** - optional approval step before downloaded tracks are finalized

### Discovery

- **Per-user recommendations** - each user's listening history shapes their own Discover page
- **Drag-and-drop layout** - customize and reorder sections per user
- **Discover playlists** - auto-generated playlists like Release Radar, adoptable into your Playlists
- **Discovery feedback** - thumbs-down artists to keep them out of your recommendations

### Playback

- **In-app audio player** - global player bar with play, pause, skip, and shuffle, streams from disk
- **Navidrome M3U path mapping** - works when Navidrome runs on a different host or filesystem
- **Plex playlist sync** - native Plexamp playlists, separate from M3U files

### Authentication

- **Reverse-proxy SSO** - pass `AUTH_PROXY_ENABLED=true` and set `x-forwarded-user` behind your proxy
- **LAN auto-login** - skip the password on your home network
- **Configurable session expiry** - `SESSION_EXPIRY_HOURS` env var (default 30 days)

### Notifications

- **Gotify** - push notifications when discovery refreshes or a flow finishes downloading
- **Webhooks** - fire HTTP requests per event with `$flowPath` and `$flowName` placeholders

### General

- **Artist pages rebuilt** - About, Similar Artists, Preview Tracks, Appears On, Release Tracks
- **Profile page** - per-user listening history provider and preferences
- **Library access test** - test from Settings that Aurral can read your Lidarr files before anything breaks
- **macOS desktop app** - thin client on GitHub Releases for Apple Silicon Macs

## Nothing to migrate

Your database upgrades to v2 automatically on first start. Your existing flows, settings, and integrations carry forward. Just update the Docker compose mount from `/app/backend/data` to `/config`, pull the new image, and restart.
