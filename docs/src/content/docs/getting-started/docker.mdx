Aurral is Docker-first. A minimal install needs two mounts: app data at `/config` and your shared media tree at `/data`.

## Minimal compose file

```yaml
services:
  aurral:
    image: ghcr.io/lklynet/aurral:latest
    restart: unless-stopped
    ports:
      - "3001:3001"
    environment:
      - PUID=${PUID:-1000}
      - PGID=${PGID:-1000}
    volumes:
      - /data:/data
      - ./config:/config
```

Change `/data:/data` to match your host media path — `/srv/media:/data` is common on NAS hardware. `./config` stores Aurral's database, settings, and encrypted integration secrets.

## Start Aurral

```bash
docker compose up -d
```

Open `http://localhost:3001` and follow onboarding.

## What the mounts do

| Mount            | Container path | Purpose                                        |
| ---------------- | -------------- | ---------------------------------------------- |
| Host media root  | `/data`        | Shared library, downloads, and generated flows |
| Host config dir  | `/config`      | Aurral database, settings, users, and jobs     |

`DOWNLOAD_FOLDER` is set during onboarding or in **Settings → Downloads → Downloads folder**. It must live inside the `/data` mount — typically `/data/downloads/aurral`.

:::note
`/config` replaces the older `/app/backend/data` path from Aurral v1. Existing databases in either location are detected automatically on startup.
:::
