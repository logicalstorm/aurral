Onboarding gets Aurral usable quickly. You can skip optional services and finish them later in Settings.

![Aurral Lidarr connection settings](../../../assets/screenshots/settings-lidarr.webp)

## Onboarding steps

1. Create your admin account.
2. Set **Downloads Folder → Path** under your Lidarr media root (for example `/data/downloads/aurral`).
3. Connect Lidarr, test library access, and choose default profiles.
4. Optionally apply the community Lidarr guide for discovery-friendly defaults.
5. Optionally connect Prowlarr (Usenet indexer), SABnzbd/NZBGet, Navidrome, Plex, Last.fm, slskd, Ticketmaster, and metadata services.

If Lidarr's stack already works, matching its media mount is the hard part. See [Match Lidarr](/getting-started/storage/).

## Good first settings

After onboarding, check these areas:

- **Settings → Lidarr**: quality profile, tag, monitoring option, and search-on-add behavior.
- **Settings → Download Clients**: Downloads Folder path, yt-dlp (default), slskd, and/or SABnzbd/NZBGet.
- **Settings → Indexers**: Prowlarr when using Usenet.
- **Settings → Playback → Plex**: optional; connect after onboarding ([Plex setup](/integrations/plex/)).
- **Settings → Discover**: Last.fm API key, refresh cadence, and discovery mode (Safer, Balanced, or Deeper).
- **Profile**: your listening-history provider (Last.fm or ListenBrainz username, or Koito instance URL).

## Per-user setup

Each user can customize their own Profile and Discover layout. Admins manage integrations, download clients, notifications, and user permissions from Settings.
