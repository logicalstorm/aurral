Onboarding is two steps: create your admin account, then connect Lidarr. Everything else lives in Settings.

![Aurral Lidarr connection settings](../../../assets/screenshots/settings-lidarr.webp)

## Onboarding steps

1. Create your admin account. Optionally enable **Auto-login on local network**.
2. Connect Lidarr (URL and API key), test the connection, then open Aurral.

If Lidarr's stack already works, matching its media mount is the hard part. See [Match Lidarr](/getting-started/storage/).

## After you land in the app

Configure these in Settings when you need them:

- **Settings → Download Clients → Downloads Folder**: path under your Lidarr media root (for example `/data/downloads/aurral`).
- **Settings → Lidarr**: quality profile, monitoring, community guide, library access check.
- **Settings → Download Clients** / **Indexers**: yt-dlp (default), slskd, Prowlarr + SABnzbd/NZBGet.
- **Settings → Playback**: Navidrome or Plex.
- **Settings → Discover**: Last.fm API key and discovery mode.
- **Profile**: listening-history provider (ListenBrainz by default until you add Last.fm or Koito).

## Per-user setup

Each user can customize their own Profile and Discover layout. Admins manage integrations, download clients, notifications, and user permissions from Settings.
