---
title: Activity
description: Track active downloads, Lidarr requests, and Aurral playlist jobs.
---

Activity replaces the older Requests and Downloads views. Use **Queue** for work in progress and **History** for completed and failed events.

![Aurral activity timeline](../../../assets/screenshots/history.webp)

Filter each view by source:

- **Lidarr** — artist and album requests, queue progress, and import status
- **slskd** — Soulseek download activity for flows and playlists
- **NZBGet** — Usenet download activity for flows and playlists (shown when Prowlarr and NZBGet are configured)
- **Aurral** — playlist generation and download jobs

Use **Queue** when you want to see what is downloading or processing right now. Use **History** when you want a chronological log of what finished or failed.

When an album request fails in Lidarr, you can retry the search from History when the entry supports it.

See [slskd](/integrations/slskd/) and [Usenet](/integrations/usenet/) for download client setup.
