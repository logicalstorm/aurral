Aurral is the discovery layer for a Lidarr-based music stack: visual, intentional, and connected to the library you already maintain.

## Main sections

### Discover

![Aurral Discover page](../../../assets/screenshots/discover.webp)

Home screen for personalized recommendations, global trends, tags, recent releases, discover playlists, recently added artists, and nearby shows. Customize which sections appear and reorder them per user.

### Search

![Aurral search results](../../../assets/screenshots/search.webp)

Find artists and albums, preview tracks, inspect library status, and add to Lidarr with your defaults.

### Library

Browse artists already in Lidarr with search, sorting, artwork, monitoring status, and full artist pages.

### Playlists

![Aurral playlists](../../../assets/screenshots/playlists.webp)

Scheduled flows, discover playlists, imported static playlists, worker settings, in-app playback, and download status. Flows regenerate on a schedule; static playlists keep fixed tracklists.

### History

![Aurral history timeline](../../../assets/screenshots/history.webp)

One timeline for Lidarr requests, slskd downloads, and Aurral playlist activity.

### Shows

![Aurral nearby shows](../../../assets/screenshots/shows.webp)

Nearby concerts for artists you care about, when Ticketmaster is configured.

### Profile

![Aurral profile listening history](../../../assets/screenshots/profile-listening.webp)

Per-user listening history (Last.fm or ListenBrainz), Lidarr defaults, and personal preferences.

### Settings

![Aurral user settings](../../../assets/screenshots/settings-user.webp)

Integrations, discovery behavior, playlist workers, notifications, users, and permissions. Admins only. Metadata providers live at **Settings → Metadata Providers**.
