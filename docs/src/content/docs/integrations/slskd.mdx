slskd handles Soulseek acquisition for flow and playlist tracks. Aurral can also use [Usenet](/integrations/usenet/) via Prowlarr and SABnzbd/NZBGet as an alternate or fallback source.

![Aurral history with slskd downloads](../../../assets/screenshots/history.webp)

## Setup

1. Run slskd with the same `/data` mount as Aurral.
2. Set slskd's download directory to `/data/downloads/slskd/complete`.
3. Connect slskd in Aurral onboarding or **Settings → Download Clients → slskd**.

Aurral picks up completed files from slskd's download path and moves them into your `DOWNLOAD_FOLDER`.

If slskd runs on Windows and reports host paths like `N:\...`, mount the parent folder into Aurral and add a path mapping under **Settings → System → Remote Path Mappings**. See [Shared storage](/getting-started/storage/#windows-and-mixed-docker-setups).

## Worker settings

Open the **slskd** card in **Settings → Download Clients** to configure:

- **Source priority** — order relative to Usenet (default `10`)
- **Preferred format** and **Strict format only** — ranking for search results
- **Clean up after runs** — clear completed slskd jobs when a flow or playlist run finishes

## Download fallback

When Usenet is also configured, Aurral tries sources in priority order. If slskd fails, Aurral can fall back to Usenet automatically. See [Usenet](/integrations/usenet/#download-source-priority-and-fallback).
