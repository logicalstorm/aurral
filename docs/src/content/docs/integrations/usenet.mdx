---
title: Usenet (Prowlarr + NZBGet)
description: Usenet-backed downloads for flows and playlists.
---

Usenet downloads use **Prowlarr** for indexer search and **NZBGet** as the download client. Aurral treats them as one download source and uses the same pipeline as slskd for flows and playlists.

Both services must be enabled and connected in **Settings → Downloads** before Usenet downloads work.

## Setup overview

1. Run Prowlarr with at least one Usenet audio indexer configured.
2. Run NZBGet with a category Aurral can use (default `aurral`).
3. Mount the same storage root into Aurral and NZBGet so Aurral can read completed files.
4. In Aurral, open **Settings → Downloads**:
   - Enable **Prowlarr**, enter URL and API key, then **Test connection** and **Refresh indexers**.
   - Enable each indexer you want Aurral to use and set indexer priority.
   - Enable **NZBGet**, enter URL and credentials, then **Test connection**.
5. Save settings.

## Prowlarr

Open the **Prowlarr** card under **Indexers**:

- **Server URL** — URL Aurral can reach (for example `http://prowlarr:9696` in Docker)
- **API key** — from Prowlarr
- **Music categories** — Newznab audio categories (default `3000`)
- **Result limit** — max releases returned per search

After **Test connection**, Aurral lists Usenet indexers from Prowlarr. Open each indexer card to:

- Enable or disable it in Aurral (independent of Prowlarr's own enable state)
- Set **Priority** — lower numbers are preferred when multiple indexers match

## NZBGet

Open the **NZBGet** card under **Download clients**:

- **Server URL** — URL Aurral can reach (for example `http://nzbget:6789` in Docker)
- **Username** and **Password**
- **Category** — NZBGet category for Aurral jobs (default `aurral`)
- **Source priority** — where Usenet sits relative to slskd (default `20`; slskd defaults to `10`)
- **NZB priority** — queue priority passed to NZBGet
- **Completed download path** — optional override if Aurral cannot infer where NZBGet stores finished files
- **Add NZBs paused** — send jobs to NZBGet in a paused state

Aurral polls NZBGet until the download completes, then imports the best matching audio file into your `DOWNLOAD_FOLDER` under `aurral-weekly-flow`.

If NZBGet runs on Windows and reports host paths such as `N:\...`, mount the parent folder into Aurral and configure path mappings under **Settings → Downloads → Path mappings**, or run **Test library access** from **Settings → Library → Lidarr**. See [Shared storage](/getting-started/storage/#windows-and-mixed-docker-setups).

## Download source priority and fallback

Aurral can run **slskd** and **Usenet** at the same time. Each client has a **source priority** — lower numbers are tried first.

When a source fails or exhausts its candidates, Aurral automatically tries the next configured source for the same track.

Example with default priorities:

1. slskd (priority `10`)
2. Usenet via NZBGet (priority `20`)

Swap priorities if you want Usenet first.

## History

When Usenet is configured, **History** shows a separate **NZBGet** tab for flow and playlist download jobs. The **slskd** tab only shows Soulseek activity.

See [History](/using/history/).
