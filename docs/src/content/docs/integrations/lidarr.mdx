Lidarr is required. Aurral uses it to add artists and albums, read library state, and surface queue and history status.

## Connection

During onboarding or in **Settings → Lidarr**, provide:

![Aurral Lidarr integration settings](../../../assets/screenshots/settings-lidarr.webp)

- The URL Aurral can reach (often `http://lidarr:8686` in Docker)
- Your Lidarr API key
- Default quality profile, metadata profile, tag, monitoring option, and search-on-add behavior
- Optional **Davo's Community Lidarr Guide** for discovery-friendly quality profiles and naming

## Library access check

Aurral tests that it can read files at the paths Lidarr reports. If this fails, confirm the shared media mount described in [Shared storage](/getting-started/storage/).

On mixed Windows/Docker setups, **Test library access** shows the exact path Lidarr reports. If that folder is mounted at a different path inside Aurral, add a mapping under **Settings → System → Remote Path Mappings**.

Path mappings let Aurral read files. If Navidrome sees the same files at a different path, enable **Use Navidrome paths in M3U files** and add a Navidrome mapping so `.m3u` files use correct paths. See [Navidrome](/integrations/navidrome/#mixed-windows-and-docker-navidrome).

## Safety

Artist and album changes go through Lidarr. Aurral does not write directly into your root music library.

## Import list feeds

Each flow can expose a Lidarr Custom List URL. Open the flow menu on **Playlists**, choose **Lidarr import URL**, then add it in Lidarr under **Settings → Import Lists → Custom List**.

The copied URL uses the same host you have open in your browser, for example `https://aurral.kell.ly/api/feeds/...`. Lidarr must be able to reach that host.

The URL includes a per-flow token. Treat it like a secret — anyone with the URL can read that flow's artist list.

The feed always reflects the flow's current tracklist.

Tracks without a MusicBrainz artist ID are omitted from the feed. Album IDs are included when Aurral has them; Lidarr uses them for **Specific Album** monitoring on versions that support album-level custom lists.
