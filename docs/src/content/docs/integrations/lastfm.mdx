Last.fm is recommended, not required. Without it, Aurral still discovers music, but recommendations, tag exploration, and discover playlists are much stronger with an API key.

## Server setup

Add your Last.fm API key in **Settings → Integrations → Last.fm** during onboarding or after install.

## Per-user listening history

Each user sets their own Last.fm username in **Profile**. On a shared Aurral instance, this keeps recommendations personal without mixing tastes together.

![Aurral profile listening history](../../../assets/screenshots/profile-listening.webp)

## Discovery settings

Admins configure refresh cadence and discovery mode (Safer, Balanced, or Deeper) in **Settings → Discover**.

If Last.fm is unavailable, Aurral can fall back to ListenBrainz for listening-history context when users have it configured.
