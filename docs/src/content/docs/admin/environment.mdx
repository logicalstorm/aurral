title: Environment variables

Most setup happens in the web UI. These variables matter for Docker deployments.

## Mounts and permissions

| Variable | Purpose |
| -------- | ------- |
| `PUID` / `PGID` | Run the container as the user that owns mounted folders. |
| `AURRAL_DATA_DIR` | Override app data directory. Default `/config`. |
| `DOWNLOAD_FOLDER` | Optional seed for the Downloads Folder path. Prefer **Settings → Download Clients → Downloads Folder → Path**. Must be absolute and under your media mount. |

## Path mappings

| Variable | Purpose |
| -------- | ------- |
| `PATH_MAPPINGS` | Translate paths for mixed Windows/Docker setups. Format `remote\|local`, separate multiples with `;`. Scope to a source with `source\|remote\|local`. Applied at runtime; edit under **Settings → Download Clients → Remote Path Mappings**. |
| `M3U_PATH_MODE` | `local` (default, Aurral paths in `.m3u` files) or `remote` (paths Navidrome can open). |
| `M3U_PATH_MAPPINGS` | Navidrome playlist path translations. Format `Aurral path\|Navidrome path`, separate multiples with `;`. Applied at runtime; not shown in the UI. |

## Authentication

| Variable | Purpose |
| -------- | ------- |
| `TRUST_PROXY` | Set when Aurral is behind a reverse proxy. |
| `AUTH_PROXY_ENABLED` | Enable reverse-proxy authentication. Default header `x-forwarded-user`. |
| `AUTH_PROXY_HEADER` | Custom header containing the authenticated username. |
| `AUTH_PROXY_DOMAIN` | Origin of your forwardAuth login page (e.g. `https://auth.example.com`). Added to the CSP `connect-src` directive so the browser doesn't block requests redirected there. |
| `AUTH_PROXY_TRUSTED_IPS` | Comma-separated proxy IP allowlist for auth headers. |
| `AUTH_PROXY_LOGOUT_URL` | Your proxy/IdP's logout endpoint (e.g. `https://auth.example.com/logout`). When set, logging out of Aurral also redirects there to end the proxy-level session, not just the local one. |
| `AUTH_PROXY_DEFAULT_ROLE` | Role assigned to proxy-auth users who aren't otherwise granted admin: `user` or `admin`. Re-evaluated on every request. |
| `AUTH_PROXY_ADMIN_USERS` | Comma-separated usernames always granted admin. Re-checked on every request, so removing a username here demotes them on their next request. |
| `AUTH_PROXY_ROLE_HEADER` | Optional header containing the user's group membership (e.g. Authelia's `Remote-Groups`), typically a comma-separated list. Checked against `AUTH_PROXY_ADMIN_GROUPS` on every request. |
| `AUTH_PROXY_ADMIN_GROUPS` | Comma-separated group names (matched against `AUTH_PROXY_ROLE_HEADER`) that grant admin. A group literally named `admin` isn't treated specially unless listed here. |
| `SESSION_EXPIRY_HOURS` | Password-login session lifetime in hours. Default `720` (30 days). |

## Logging

| Variable | Purpose |
| -------- | ------- |
| `AURRAL_VERBOSE_LOGS` | Fuller server logs while troubleshooting. |

## Last.fm tuning

| Variable | Purpose |
| -------- | ------- |
| `AURRAL_LASTFM_TIMEOUT_MS` | Per-request timeout for Last.fm API calls in milliseconds. Default `15000`. Increase if focus playlists are missing tracks and logs show `ECONNABORTED` errors. |

## Spotify playlist import

Optional overrides for the shared Lidarr Spotify OAuth proxy. Defaults work for most installs.

| Variable | Purpose |
| -------- | ------- |
| `SPOTIFY_CLIENT_ID` | OAuth client ID. |
| `SPOTIFY_OAUTH_REDIRECT_URI` | Authorization redirect. Default `https://spotify.lidarr.audio/auth`. |
| `SPOTIFY_OAUTH_RENEW_URI` | Token refresh endpoint. Default `https://spotify.lidarr.audio/renew`. |

## Example

```yaml
environment:
  - PUID=1000
  - PGID=1000
  - TRUST_PROXY=true
```

For mounts and path mappings, see [Match Lidarr](/getting-started/storage/).
