---
title: Storage & backups
description: App data, media mounts, and what to back up.
---

Aurral keeps app state and generated music separate.

For mounts, follow [Match Lidarr](/getting-started/storage/): same media root as Lidarr, downloads path set in the UI.

## What lives where

| Path                             | Contains                                                                                                                                                                   |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/config`                        | Database, settings, users, sessions, cache state, and playlist jobs. Mount with `./config:/config`; not shared with other apps.                                            |
| `/data` (or Lidarr's media root) | Shared library and downloads. Same mount Lidarr uses.                                                                                                                      |
| Downloads Folder                 | Generated flows, imported playlists, staging, and playlist files. Set in **Settings → Download Clients → Downloads Folder → Path** (for example `/data/downloads/aurral`). |

:::note
`/config` replaces the older `/app/backend/data` path from Aurral v1. Existing databases in either location are detected automatically on startup.
:::

## Back up

- Your mounted `/config` folder
- The host folder behind your Downloads Folder path if you want generated playlists to survive rebuilds
- Your compose file and `.env`

## Library safety

- Aurral does not write into your root music library.
- Artist and album changes go through Lidarr.
- Flows and imported playlists stay under Aurral's Downloads Folder path.
